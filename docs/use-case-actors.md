# Диаграмма акторов и прецедентов (Use Case)

Сохраните: `docs/images/use-case.png`

---

## Акторы (человечки сбоку)

| Актор | Описание |
|-------|----------|
| **Гость** | Не залогинен |
| **Пользователь (USER)** | Зарегистрированный пользователь |
| **Администратор (ADMIN)** | USER + управление пользователями (минимум: список users, опционально блокировка) |
| **Владелец кошелька** | USER с ролью WALLET_OWNER в конкретном Wallet — *вторичный актор* |
| **Участник кошелька** | USER с ролью WALLET_MEMBER — *вторичный актор* |
| **Система (планировщик)** | Внутренний актор для `@Scheduled` recurring |

Связи:

- `USER` — обобщение: обычный пользователь может стать Владельцем или Участником в разных кошельках.  
- На диаграмме можно: один актор «Пользователь», а внутри овала прецедента помечать «только Owner».

## Прецеденты внутри границы системы «Трекер расходов»

### Аутентификация (Гость / USER)

| ID | Прецедент |
|----|-----------|
| UC-01 | Зарегистрироваться |
| UC-02 | Войти (получить JWT) |
| UC-03 | Выйти |

### Кошелёк (USER → Owner / Member)

| ID | Прецедент | Кто |
|----|-----------|-----|
| UC-10 | Создать кошелёк | USER (становится Owner) |
| UC-11 | Пригласить участника | Owner |
| UC-12 | Просмотреть список своих кошельков | USER |
| UC-13 | Просмотреть баланс кошелька | Owner, Member |
| UC-14 | Удалить/изменить кошелёк | Owner |

### Категории

| ID | Прецедент | Кто |
|----|-----------|-----|
| UC-20 | Создать категорию | Owner |
| UC-21 | Изменить/удалить категорию | Owner |
| UC-22 | Просмотреть категории | Owner, Member |

### Операции

| ID | Прецедент | Кто |
|----|-----------|-----|
| UC-30 | Добавить доход (INCOME) | Owner, Member |
| UC-31 | Добавить расход (EXPENSE) | Owner, Member |
| UC-32 | Изменить свою операцию | Member (только свои), Owner (любые) |
| UC-33 | Удалить операцию | то же |
| UC-34 | Разделить расход (split, фикс. суммы) | Owner, Member |

### Бюджет и отчёты

| ID | Прецедент | Кто |
|----|-----------|-----|
| UC-40 | Задать бюджет на период | Owner |
| UC-41 | Просмотреть отклонение от бюджета | Owner (Member — опционально только просмотр) |
| UC-42 | Сформировать отчёт за период | Owner, Member |

### Регулярные платежи

| ID | Прецедент | Кто |
|----|-----------|-----|
| UC-50 | Создать правило «раз в месяц» | Owner (или Member — если решите иначе) |
| UC-51 | Система создаёт операцию по расписанию | Система (планировщик) |

### Админ

| ID | Прецедент | Кто |
|----|-----------|-----|
| UC-60 | Просмотреть всех пользователей | ADMIN |

## Связи include / extend (по желанию, 2 штуки достаточно)

- `UC-30` и `UC-31` **include** «Выбрать категорию»  
- `UC-02` **include** «Проверить учётные данные»  
- `UC-34` **extend** `UC-31` (split — особый случай расхода)

```
        [Гость]                    [ADMIN]
           |                            |
    UC-01, UC-02                   UC-60
           |                            |
           +------------+---------------+
                        |
                   [ USER ]
                        |
         +--------------+--------------+
         |                             |
    [Владелец кошелька]          [Участник]
         |                             |
    UC-11,14,20,40                  UC-30,31,32
    UC-32 (все)                     UC-42,13

              [ Система (планировщик) ]
                        |
                     UC-51
```

Граница системы — прямоугольник вокруг всех UC-xx.
